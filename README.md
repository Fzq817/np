# GAT-PPO for Multi-objective DHDNPFSP

This repository provides the core algorithm and instance-generation code for the paper:

**Graph Attention Network-based Proximal Policy Optimization for Multi-objective Distributed Heterogeneous Dynamic Non-permutation Flow-shop Scheduling Problem**

---

## Overview

This code implements a GAT-PPO-based deep reinforcement learning framework for solving the multi-objective distributed heterogeneous dynamic non-permutation flow-shop scheduling problem (DHDNPFSP).

The repository includes:

- The core GAT-PPO algorithm
- Preference-conditioned actor-critic network
- Pareto archive management
- Preference-Guided Posterior Selection (PGPS)
- Unified instance generator
- Training, testing, and evaluation scripts

---

## Code Structure

| File | Description |
|---|---|
| `models.py` | Heterogeneous preference-conditioned GAT encoder and actor-critic network |
| `trainer.py` | PPO training loop with external Pareto archive update |
| `pareto_pref.py` | Preference sampler and Pareto archive management |
| `pgps.py` | Preference-Guided Posterior Selection (PGPS) inference |
| `evaluation.py` | Multi-preference deterministic evaluation |
| `generate_eval_instances.py` | Unified instance generator for all algorithms |
| `batch_generate.py` | Batch wrapper for generating instance sets by seed range |
| `train.py` | Training entry point |
| `test.py` | Test evaluation entry point |
| `pgps_eval.py` | PGPS inference entry point |
| `main.py` | CLI dispatcher |

---

## Train / Validation / Test Splits

All algorithms use the same instance files generated by `generate_eval_instances.py`.

The generated instances are based on the Demirkol benchmark and are extended with:

- Factory heterogeneity
- Energy-consumption parameters
- Dynamic priority-driven job insertion

The random seed convention is as follows:

| Split | Seeds |
|---|---|
| Train | 1–100 |
| Validation | 101–200 |
| Test | 201–300 |

## Experiment Details

### Training Hyperparameters

The training hyperparameters were selected using the Taguchi L16 orthogonal array.

| Parameter | Value |
|---|---:|
| Number of episodes | 1000 |
| Max steps per episode / rollout length | 400 |
| Batch size `B` | 128 |
| PPO epochs per update | 6 |
| Learning rate `α` | 5e-5 |
| Discount factor `γ` | 0.98 |
| Entropy coefficient `c₂` | 0.01 |
| GAT layers `L` | 2 |
| Attention heads | 4 |
| Clip epsilon | 0.08 |

### Evaluation and Reproducibility Settings

| Parameter | Value |
|---|---|
| Evaluation frequency | Every 50 episodes |
| Evaluation preference vectors | 7 fixed vectors, including extreme and interior directions |
| Training random seeds | 42, 43, 44 |
| Independent runs | 3 runs per problem class |
| Test realization seed | 12345 |
| Archive dominance check interval | Every 10 episodes |

---

## PGPS Inference Settings

The PGPS inference mechanism is used to select a posterior solution according to the user-specified preference or reference requirement.

| Parameter | Value |
|---|---|
| Candidate trajectories `N` | 16 by default |
| Sensitivity analysis of `N` | 1, 2, 4, 8, 16, 32, 64 |
| Steering coefficient `β` | 1.0 |

## Usage

### Generate Instances

```bash
python generate_eval_instances.py
```

For batch generation over multiple seed ranges:

```bash
python batch_generate.py
```

### Train GAT-PPO

```bash
python train.py
```

### Test the Trained Model

```bash
python test.py
```

### Run PGPS Inference

```bash
python pgps_eval.py
```

### Use the CLI Dispatcher

```bash
python main.py
```
